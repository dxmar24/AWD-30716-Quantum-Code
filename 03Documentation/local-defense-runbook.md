# Local Defense Runbook

Use this sequence before the defense. It keeps AWS stopped and runs the complete project locally.

## 1. Prerequisites

- Docker Desktop is running.
- Node.js is version 20.19 or newer.
- Python virtual environment exists at `06Code/apis/python-analytics-api/.venv`.
- Ignored `06Code/.env` contains local PostgreSQL, session and seed values.
- Never project the contents of `.env` or `07Other/local-secrets/`.

## 2. Start PostgreSQL And The Local Mailbox

From `06Code`:

```powershell
npm run db:local:up
npm run db:migrate:status
npm run db:migrate
```

Load the realistic and repeatable defense data only when the database needs to be rebuilt:

```powershell
npm run db:seed:academic-demo
```

The seed is idempotent and supplies five branches, 25 named students, 11 teachers, 11 dance groups, finalized and upcoming sessions, attendance, payments, events and enrollment prospects.

The same command starts Mailpit, the safe local SMTP inbox used to demonstrate account invitations without sending test messages to the public internet:

- Mailbox: `http://127.0.0.1:8025`
- SMTP receiver used by Node: `127.0.0.1:1025`

Keep Mailpit open during the account demo. Create or resend an invitation in the General Director panel, then open the newest message to show the branded email and temporary password. The director dashboard and API response must never display that password.

## 3. Start Node, React And Express

Open a terminal in `06Code`:

```powershell
npm start
```

`npm start` first builds React with Vite and then starts Express. With the current local environment, open:

- Academy and login: `http://127.0.0.1:3005`
- Node health: `http://127.0.0.1:3005/api/v1/health/ready`

The standard default is port `3000`; the active `.env` may override it.

## 4. Start Python Analytics

Open a second terminal:

```powershell
cd 06Code/apis/python-analytics-api
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8005
```

Verify `http://127.0.0.1:8005/api/analytics/v1/health`.

## 6. Visual Database Access

Open DBeaver or pgAdmin and create one saved local connection:

| Field | Value |
| --- | --- |
| Host | `127.0.0.1` |
| Port | `5432` |
| Database | `american_latin_class` |
| User and password | Read the ignored local credentials file |

After saving the connection, future access only requires Docker Desktop and the PostgreSQL container to be running. Expand Schemas, `public`, Tables, then right-click a table and choose View Data. Useful tables are `users`, `students`, `class_groups`, `class_sessions`, `student_attendance_records`, `student_payments` and `audit_logs`.

## 7. Postman

Import the environment and both collections listed in `postman/README.md`. Select the local environment, add the login password only in Postman's current value, then run the main collection followed by the analytics collection.

## 9. Docker And WSL Recovery

If Docker Desktop reports `DockerDesktop/WSL/CommandTimeout` while listing distributions, do not reset Docker or delete its data. Quit Docker Desktop, open PowerShell as Administrator and run:

```powershell
taskkill /F /IM wslservice.exe
sc.exe start WslService
wsl.exe --list --verbose --all
```

Both Ubuntu distributions may remain stopped; the important result is that the list returns instead of timing out. Start Docker Desktop again and wait until the engine is ready. The academy database uses the preserved `proyecto-danza_danza_pgdata` volume and the `proyecto-danza-postgres` container exposes PostgreSQL only on `127.0.0.1:5432`.

A verified custom-format backup is stored locally at `07Other/local-secrets/american-latin-class-defense-final.dump`. The folder is ignored by Git because the database may contain private data.

## 10. Cost Control

Keep all five EC2 instances stopped until the final deployment window. Starting an EC2 instance can also expose changing public IPv4 addresses, so do not spend credit merely to practice. Local verification exercises the same source code, PostgreSQL schema and authorization rules.
