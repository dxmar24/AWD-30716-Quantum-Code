# Local Demo Data and Database Inspection

Demo seeds are intentionally restricted to `NODE_ENV=development`, an explicitly opted-in PostgreSQL database on localhost, and externally supplied credentials. They cannot run in test, staging, production, or against a remote host.

```bash
cd 06Code
NODE_ENV=development ALLOW_LOCAL_DEMO_SEEDS=true npm run db:seed:academic-demo
```

Before running the command, provide every required `SEED_*` value through an ignored local `.env` or secret manager. No seed contains a default password, and seed output never prints passwords. All seeded password accounts are marked `mustChangePassword=true`.

The seed supplies branches, students, teachers, groups, sessions, enrollments, payments, attendance and events suitable for local reports. Addresses generated by the seed use the reserved `example.invalid` domain unless a privileged local account address is supplied explicitly.

The local PostgreSQL container binds only to `127.0.0.1:5432`. Use DBeaver, pgAdmin, or another client through that loopback address. Production inspection follows the private SSM/VPN procedure in `03Documentation/cloud-database-viewer.md`; no browser database viewer is published.

Never commit populated environment files, database exports, connection profiles, screenshots with personal data, hostnames, tokens, or credentials.
