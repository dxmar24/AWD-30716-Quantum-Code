# American Latin Class System - Project Definition

## Problem statement
American Latin Class operates five branches with manual and inconsistent attendance processes. Directors lack centralized, traceable and current data about students, teachers, class attendance, teacher hours, scholarship candidates, level promotion candidates and branch performance.

## General objective
Build a professional academic web system focused on attendance control and academic tracking for American Latin Class.

## Specific objectives
- Centralize student attendance and teacher check-in/check-out.
- Identify scholarship candidates when two-month attendance reaches at least 90%.
- Support B1 to B2 promotion evaluations with attendance and theory/practice evidence.
- Protect private operations with Google OAuth, app-owned roles and secure sessions.
- Produce branch and consolidated reports for directors.

## Scope
Public landing page, enrollment requests, backend APIs, PostgreSQL schema, tests, AWS deployment documentation and UML scripts.

## Out of scope
Automatic scholarship approval, payroll bank integrations, payment gateway processing and mobile native apps.

## Stakeholders
Visitors, students, teachers, branch directors, general director, administrator and academic reviewers.
